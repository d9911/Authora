> Date: 2026-07-11

## Audit result

## Excluded aggregate or duplicate entry points

- `backend-test` and `backend-test-sqlite` are exact aliases; the runner executes their three leaves once.
- `check`, `check-types`, and `check-account-recovery` are compatibility aggregators; their leaves are run directly.
- backend/frontend `lint` repeat boundary/source checks and TypeScript checks already in the catalog.
- `check-source` remains compatible but its 28 leaves are run independently so one failure cannot suppress the remaining source checks.
- `check-i18n-http` aggregates the production fallback and an external-base HTTP check; the runner executes the fallback once and owns a managed frontend for its HTTP leaf.
- `test-all` only aggregates security and load; those leaves are run directly.
- backend build is executed once and reused by recovery, refresh, security, and load checks.
- `frontend/e2e-test.mjs` is recorded as non-applicable legacy coverage because it still uses an undeclared Mongo dependency, fixed ports, and pre-localization routes; it is not reported as a successful runner step.

## Implementation sequence

1. Add failing `node:test` coverage for sequencing, statuses, continuation, timers, color policy, JSON output, exit-code derivation, signals, deduplication, and path containment.
2. Implement a dependency-free core runner using `child_process`, `fs`, `path`, `process`, and `perf_hooks`.
3. Add the declarative project catalog and keep `check-source.mjs` backed by the same source-check registry.
4. Add managed frontend/backend helpers for i18n HTTP and optional load leaves, with readiness, timeout, and guaranteed cleanup.
5. Make the security/load/refresh scripts propagate real failures and clean up only processes they started.
6. Add `make test`, preserve all old targets, ignore only `.test-results/`, and document modes/statuses/dependencies.
7. Run runner tests, full plain suite, TTY/color evidence, verbose fixture evidence, production build/runtime checks, `git diff --check`, and final status/diff review.
