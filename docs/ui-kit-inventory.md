## Confirmed audit findings

1. `ModalMain.tsx` renders inline-styled `div` elements and never imports its existing SCSS module; runtime inspection found zero `[role="dialog"]`, zero `[aria-modal="true"]`, unchanged body overflow, focus on the launcher, and no Escape close.
2. `Tabs.tsx` has `tablist/tab` roles but no roving `tabIndex`, Arrow/Home/End handling, `aria-controls`, or panel relationship; runtime ArrowRight did not move focus or selection.
3. `HeaderMain.tsx` implements local outside-click/Escape behavior, but no menu-item Arrow/Home/End navigation or focus restoration.
4. `ProfilePhotoManager.tsx` uses `window.confirm`, so confirmation does not use Identity Aura theming and cannot expose busy state.
5. Repeated avatar image/fallback markup exists in `HeaderMain.tsx` and `widgets/ProfileCard/ProfileCard.tsx`; repeated verification tags duplicate `Badge`.
6. Calculated WCAG contrast ratios for confirmed small-text pairs include: white on light `--iris` (`#5b4bff`) = 5.39:1 (passes), white on dark `--iris` (`#8b7cff`) = 3.27:1 (fails), white on light `--alert` = 3.11:1 (fails), and light `--signal` on white = 2.55:1 (fails). Values were computed from the hex tokens in `globals.scss` using WCAG relative luminance `(Lmax + 0.05) / (Lmin + 0.05)` without rounding the pass/fail decision.
